# Двухступенчатая скоринговая система

## Описание проекта

Проект представляет собой двухступенчатую скоринговую систему, разработанную на основе датасета [Home Credit Default Risk](https://www.kaggle.com/c/home-credit-default-risk) от Home Credit. Система предназначена для оценки кредитоспособности клиентов и прогнозирования риска дефолта при выдаче кредита.

Основная цель — автоматизировать процесс принятия решений о выдаче кредита с помощью машинного обучения, используя как общие признаки клиента, так и исторические данные из бюро кредитных историй (bureau data).

## Применяемые модели

Система использует два уровня моделей, что обеспечивает гибкость и точность предсказаний:

### 1. Первая модель (`first_best`)
- **Назначение**: Базовая оценка на основе признаков, заполненных вручную (анкетные данные).
- **Тип модели**: Могут использоваться различные модели (CatBoost, XGBoost, Torch-MLP), выбор зависит от результатов обучения. В нашем случае - CatBoost
- **Данные**: Только входные признаки из формы.
- **Использование**: Когда недоступен идентификатор клиента (`sk_id_curr`) или нет данных в бюро.

### 2. Вторая модель (`second_best`)
- **Назначение**: Уточнённая оценка на основе данных из бюро кредитных историй.
- **Тип модели**: Аналогично первой — CatBoost, XGBoost или Torch-MLP. В нашем случае - XGBoost
- **Данные**: Автоматически подгружаются из таблицы `bureau` по `sk_id_curr`.
- **Использование**: Когда пользователь указывает `sk_id_curr` и соответствующая запись существует в базе.

### Логика выбора модели
- Если передан `sk_id_curr` → пытаемся получить данные из `bureau` → используем **вторую модель**.
- Если `sk_id_curr` не передан, но есть признаки → используем **первую модель**.
- Если `sk_id_curr` передан, но нет в `bureau` → ошибка "Такого id нет, заполните поля вручную".

## Архитектура проекта
<img width="871" height="471" alt="architechture" src="https://github.com/user-attachments/assets/40a571b8-45c2-4e8e-96d8-1346a6358b6d" />
Проект построен по модульной архитектуре с разделением на три основные части:

### 1. Бэкенд (`src/backend`)
- **Фреймворк**: FastAPI.
- **Функции**:
  - Загрузка и инициализация моделей.
  - Подключение к PostgreSQL.
  - Предсказание вероятности дефолта.
  - Автозаполнение признаков по `sk_id_curr`.
- **Маршруты**:
  - `GET /schema` — возвращает список ожидаемых признаков.
  - `GET /prefill/{sk_id}` — возвращает признаки из `bureau`.
  - `POST /predict` — выполняет предсказание.
  - `GET /health` — проверка работоспособности.

### 2. Фронтенд (`src/frontend`)
- **Фреймворк**: Streamlit.
- **Функции**:
  - Визуальный интерфейс для заполнения признаков.
  - Поддержка двух режимов ввода: ручной ввод или по `sk_id_curr`.
  - Отображение результата предсказания (скор, вероятности).

### 3. Инфраструктура
- **Docker** — контейнеризация сервисов.
- **Docker Compose** — оркестрация запуска:
  - `postgres`: хранение данных из `bureau.csv`.
  - `data-loader`: загрузка данных в БД перед запуском бэкенда.
  - `backend`: API-сервис на FastAPI.
  - `frontend`: веб-интерфейс на Streamlit.
- **Скрипт загрузки**: `load_bureau.py` — загружает `bureau.csv` в таблицу `bureau`.

## Установка и запуск
В первую очередь клонируйте проект
```
git clone https://github.com/Artem1s1337/diplom_credit_scoring.git
```
После клонирования репозитория создайте внутри проекта папку `/data` и сохраните туда файлы с Яндекс.Диска: https://disk.yandex.ru/d/tJfrSwyEtuF7_g. Не переименовывайте их, это данные моделей:
```
# Переходим в директорию
cd diplom_credit_scoring

# Создаём папку
mkdir data
```
### Требования
- Python 3.12
- [uv](https://docs.astral.sh/uv/) — современный пакетный менеджер для Python.
- Docker и Docker Compose.

### Установка зависимостей

#### Windows

```powershell
# Установка uv
pip install uv

# Создание виртуального окружения
uv venv

# Активация окружения
Set-ExecutionPolicy -ExecutionPolicy Bypass -Scope Process
.\.venv\Scripts\Activate.ps1

# Установка зависимостей
uv add -r requirements.txt
```

#### Linux / macOS

```bash
# Установка uv
pip install uv

# Создание виртуального окружения
uv venv

# Активация окружения
source .venv/bin/activate

# Установка зависимостей
uv add -r requirements.txt
```

### Запуск проекта

Проект запускается через Docker Compose. Порядок:
1. Запуск PostgreSQL.
2. Выполнение `load_bureau.py` для загрузки данных.
3. Запуск бэкенда.
4. Запуск фронтенда.

#### Команда запуска

```bash
# Запускаем в фоновом режиме
docker-compose up -d --build
```

После запуска:
- Бэкенд будет доступен по адресу: `http://localhost:8000`
- Документация API: `http://localhost:8000/docs`
- Фронтенд: `http://localhost:8501`

### Остановка

```bash
docker-compose down
```

## Переменные окружения

Конфигурация задаётся через файл `.env`. Пример в `.env.example`:

```env
POSTGRES_HOST=
POSTGRES_PORT=
POSTGRES_DB=
POSTGRES_USER=
POSTGRES_PASSWORD=
DATABASE_URL=postgresql://user:password@host:port/db
BACKEND_URL=http://backend:8000
```

> **Примечание**: Внутри Docker контейнеров `POSTGRES_HOST` автоматически переопределяется как `postgres`.

## Troubleshooting

При возникновении проблем проверьте:
- Доступность портов 8000, 8501, 5432.
- Наличие `bureau.csv` в `data/`.
- Корректность `.env` файла.
